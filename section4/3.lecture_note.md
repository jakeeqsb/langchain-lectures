# LangChain 강의 노트 — Section 4-3: LangChain v1 `create_agent` 마이그레이션

> **강의 주제**: `create_react_agent` + `AgentExecutor` → LangChain v1 `create_agent`로의 마이그레이션
> **난이도**: 중급
> **사용 패키지**: `langchain`, `langchain-openai`, `langchain-tavily`, `pydantic`, `python-dotenv`

```bash
pip install langchain langchain-openai langchain-tavily pydantic python-dotenv
```

> ⚠️ **버전 주의**: `create_agent`는 **LangChain v1 이상** (LangGraph 기반) 에서 사용 가능.
> `langchain_classic` 패키지(구버전)와 신버전을 혼용하지 않도록 주의.

---

## 1. 구버전 → 신버전 마이그레이션: 무엇이 사라졌나

### 구버전 (`create_react_agent` + `AgentExecutor` 조합)

```python
# ❌ 구버전 코드 — LangChain v0.x / langchain_classic 스타일
from langchain_classic import hub
from langchain_classic.agents import AgentExecutor
from langchain_classic.agents.react.agent import create_react_agent
from langchain_core.prompts import PromptTemplate
from langchain_core.runnables import RunnableLambda
from langchain_openai import ChatOpenAI
from langchain_tavily import TavilySearch
from schemas import AgentResponse

tools = [TavilySearch()]
llm = ChatOpenAI(model="gpt-4", temperature=0)
structured_llm = llm.with_structured_output(AgentResponse)

# ReAct 프롬프트 + Format Instructions 수동 조합
react_prompt = hub.pull("hwchase17/react")
react_prompt_with_format = PromptTemplate(
    template=REACT_PROMPT_WITH_FORMAT_INSTRUCTIONS,
    input_variables=["input", "agent_scratchpad", "tool_names"],
).partial(format_instructions="")

# 추론 체인 + 실행기 + 출력 파이프라인 수동 조립
agent = create_react_agent(llm=llm, tools=tools, prompt=react_prompt_with_format)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
extract_output = RunnableLambda(lambda x: x["output"])
chain = agent_executor | extract_output | structured_llm

# 실행
result = chain.invoke({"input": "search for AI engineer jobs..."})
print(result)
```

### 신버전 (`create_agent` — LangChain v1 / LangGraph 기반)

```python
# ✅ 신버전 코드 — LangChain v1 스타일
from langchain.agents import create_agent
from langchain_openai import ChatOpenAI
from langchain_tavily import TavilySearch
from schemas import AgentResponse

tools = [TavilySearch()]
llm = ChatOpenAI(model="gpt-4o", temperature=0)

# 한 줄로 에이전트 생성
agent = create_agent(
    model=llm,
    tools=tools,
    response_format=AgentResponse,  # Pydantic 구조화 출력 직접 지정
)

# 입력 형식 변경: "input" 키 → messages 리스트
result = agent.invoke({
    "messages": [
        {
            "role": "user",
            "content": "search for 3 job postings for an AI engineer using LangChain in the Bay Area",
        }
    ]
})

# 구조화 응답 접근 키도 변경
structured = result.get("structured_response", None)
print(structured if structured is not None else result)
```

### 제거된 코드 비교

| 구버전 요소 | 제거 이유 |
|------------|----------|
| `hub.pull("hwchase17/react")` | ReAct 프롬프트 불필요 — Function Calling으로 추론 |
| `PromptTemplate` + `partial()` | 커스텀 ReAct 프롬프트 자체가 사라짐 |
| `AgentExecutor` | LangGraph 내부에서 자동 처리 |
| `RunnableLambda(lambda x: x["output"])` | `structured_response` 키로 직접 접근 |
| `with_structured_output()` (파이프 조합) | `response_format` 인자로 대체 |
| LCEL `|` 파이프 조립 전체 | 내부에서 자동 관리 |

> 🔍 **내부 동작 원리**: `create_agent`는 LangGraph 위에서 동작한다
>
> ```text
> 구버전 AgentExecutor:
>   while True:
>     response = llm.invoke(messages)      # 개발자가 루프 직접 관리
>     if tool_calls: execute → append
>     else: return
>
> 신버전 create_agent (LangGraph StateGraph):
>   START → agent_node (LLM 추론)
>             ↓ tool_calls 있으면
>           tool_node (Tool 실행)
>             ↓ 결과를 state에 추가
>           agent_node (LLM 재호출)
>             ↓ tool_calls 없으면
>           END → structured_response 반환
>
>   → AgentExecutor while 루프, extract_output lambda,
>     LCEL 파이프가 모두 LangGraph 내부로 흡수됨
> ```

> 💡 **실무 포인트**: 신버전으로 마이그레이션하면 코드량이 절반 이하로 줄어든다.
> 단, 내부가 LangGraph 기반으로 바뀌어 디버깅 시 LangSmith에서
> **Graph 실행 흐름** 형태로 확인해야 한다 (AgentExecutor 체인 로그와 다름).

🔗 [create_agent 공식 문서](https://python.langchain.com/docs/concepts/agents/)
🔗 [LangGraph 공식 문서](https://langchain-ai.github.io/langgraph/)

## 2. 입력 형식 변경: `"input"` 키 → `messages` 리스트

### 구버전

```python
# 구버전: "input" 키에 문자열 전달
result = chain.invoke({"input": "search for AI engineer jobs..."})
```

### 신버전

```python
# 신버전: OpenAI Chat 형식과 동일한 messages 리스트
result = agent.invoke({
    "messages": [
        {"role": "user", "content": "search for AI engineer jobs..."}
    ]
})
```

> 🔍 **내부 동작 원리**: 입력이 `messages` 리스트로 바뀐 이유
>
> ```text
> 구버전 AgentExecutor:
>   "input" 문자열 → 내부에서 HumanMessage로 변환 후 처리
>   → 추상화 레이어가 입력 형식을 숨김
>
> 신버전 create_agent (LangGraph):
>   StateGraph의 state가 처음부터 messages 리스트
>   → 각 노드(agent, tool)가 messages를 직접 읽고 씀
>   → 입력 형식이 내부 state 구조와 그대로 일치
>
>   이 변화는 LangGraph MessagesState를 직접 노출한 것
>   → 멀티턴 대화, Human-in-the-loop, 메모리 관리가 훨씬 유연해짐
> ```

> 💡 **실무 포인트**: `messages` 형식은 LangGraph의 `MessagesState`와 동일하다.
> LangGraph를 직접 사용할 때도 이 패턴이 그대로 이어지므로 익숙해두면 좋다.

## 3. 구조화 출력: `response_format` 인자

### 구버전 — LCEL 3단 파이프라인

```python
# ❌ 구버전: 3단계 파이프라인 수동 조립
structured_llm = llm.with_structured_output(AgentResponse)
extract_output = RunnableLambda(lambda x: x["output"])
chain = agent_executor | extract_output | structured_llm

result = chain.invoke({"input": "..."})
# 결과: structured_llm에 의해 AgentResponse 객체 반환
```

### 신버전 — `response_format` 인자 한 줄

```python
# ✅ 신버전: create_agent 인자로 직접 지정
agent = create_agent(
    model=llm,
    tools=tools,
    response_format=AgentResponse,   # ← 이 한 줄이 전부
)
result = agent.invoke({"messages": [...]})

# ⚠️ 접근 키 변경: result["output"] → result["structured_response"]
structured = result.get("structured_response")  # Pydantic AgentResponse 객체
```

> 🔍 **내부 동작 원리**: `response_format` 처리 흐름
>
> ```text
> response_format=AgentResponse 지정 시 내부 처리:
>   1. Final Answer 단계에서 llm.with_structured_output(AgentResponse) 자동 적용
>   2. Function Calling API로 AgentResponse 스키마 강제
>   3. LLM 응답 JSON → AgentResponse Pydantic 객체 자동 변환
>   4. result["structured_response"] 에 저장
>
>   → extract_output lambda, parse, with_structured_output 파이프 조합 불필요
> ```

> ⚠️ **출력 접근 키 변경**: `result["output"]` (구버전) → `result["structured_response"]` (신버전).
> 마이그레이션 시 이 키 변경을 빠뜨리면 `None`이 반환된다.

## 4. AI 코드 에디터를 활용한 마이그레이션

강의에서는 **Cursor Agent**를 이용해 마이그레이션을 수행했다. 핵심 워크플로우:

```text
1. LangChain 공식 문서 → "Copy as text" 버튼으로 docs 텍스트 복사
   (llm.txt 형식 — LLM이 읽기 좋은 구조화 문서)

2. Cursor에 프롬프트:
   "Migrate the code in main.py to the newest create_agent interface.
    Here are the docs: [복사한 내용 붙여넣기]"

3. 생성된 diff 검토:
   - 제거: hub.pull, PromptTemplate, AgentExecutor, RunnableLambda, LCEL 파이프
   - 추가: create_agent, response_format, messages 입력 형식, structured_response 접근
```

> 💡 **실무 포인트**: LangChain처럼 업데이트가 잦은 라이브러리는
> 공식 문서를 컨텍스트로 제공해야 LLM이 정확한 신버전 코드를 생성한다.
> AI 에디터로 레거시 코드를 마이그레이션할 때의 표준 워크플로우다.

## 5. LangChain 에이전트 버전 진화 전체 흐름

```text
Layer 3 → ReAct 프롬프트 기반 (텍스트 파싱)
  create_react_agent → "Thought:/Action:/Observation:" 텍스트 파싱
  문제: 파싱 오류 빈번, 소형 모델에서 불안정

Layer 2 → Tool Calling Agent
  create_tool_calling_agent → Function Calling API로 Tool 선택
  개선: 신뢰성 향상 / 구조화 출력은 여전히 수동 파이프 조립 필요

Layer 1 → with_structured_output + LCEL 파이프
  agent_executor | extract_output | structured_llm
  개선: 구조화 출력 통합 / 하지만 파이프라인 수동 조립 코드 필요

Layer 0 → create_agent (LangChain v1 / LangGraph)
  create_agent(model, tools, response_format=...)
  개선: 모든 것이 LangGraph 내부로 흡수 / 단순한 인터페이스
  내부: LangGraph StateGraph로 동작

레거시 레이어를 배우는 이유:
  → "create_agent" 한 줄이 편리하지만,
    내부 원리를 모르면 프로덕션 오류 앞에서 속수무책
  → while 루프 → AgentExecutor → LangGraph 흐름을 알아야
    어떤 오류든 원인을 역추적할 수 있다
```

> 💡 **실무 포인트**: 추상화 레이어가 높을수록 편리하지만 디버깅이 어려워진다.
> 프로덕션에서 LangSmith 트레이스가 필수인 이유가 바로 이것이다.

🔗 [LangChain 에이전트 개념 변화](https://python.langchain.com/docs/concepts/agents/)
🔗 [ReAct 논문 (배경 이해)](https://arxiv.org/abs/2210.03629)
